《代码大全》总结

发表于 2023-07-18

三条理念：抽象是应对复杂性的核心利器，软件构建是一项人的协作活动，始终面向问题来进行软件构建。

《代码大全》是一本指导程序员如何成为一流，以及如何更快更好地进行软件构建的书籍。为达成这一目标，作者在本书中讨论了关于软件构建方方面面的问题，使读者可以从更广更全面的视角来看待软件构建，以培养出成为一流程序员所需的思维习惯。与此同时，作者并非如空中楼阁一般高谈阔论，而是深入到具体的编程实践中，为读者指明向一流程序员迈步的具体措施。此外，软件构建牵扯到很多复杂的领域，作者难以在书中深入探讨，但他在相关章节后给出了更多的参考资源，为读者更进一步的了解和学习指出了方向。

完成一次优秀的软件构建需要关注方方面面的问题，大到确定需求和软件架构，小到选定变量名和控制语句，此外还包括版本管理、协同构建、测试等等问题，不一而足。这些问题都具有一定的复杂性，一流程序员能够顺利解决各式各样的问题，高效地构建出优质的软件。要达成这一目标，核心就是降低软件构建过程中的复杂性，以免陷入复杂的泥沼，导致项目超时延期，bug频出，甚至于构建失败，项目流产。

对一本书进行总结概括需要找到恰当的出发点，与对软件整体架构进行描述如出一辙。对于《代码大全》这本书来说，我认为可以用三条理念来总体把握，即抽象是应对复杂性的核心利器，软件构建是一项人的协作活动，以及，始终面向问题来进行软件构建。用这三条理念来指导和审视自己的代码构建的整个过程，相信能够让我的代码构建质量更上一层楼。
