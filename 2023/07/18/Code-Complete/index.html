《代码大全》总结

发表于 2023-07-18 更新于 2023-07-20

三条理念：抽象是应对复杂性的核心利器，始终面向问题来进行软件构建，软件构建是一项人的协作活动。

《代码大全》是一本指导程序员如何成为一流，以及如何更快更好地进行软件构建的书籍。为达成这一目标，作者在本书中讨论了关于软件构建方方面面的问题，使读者可以从更广更全面的视角来看待软件构建，以培养出成为一流程序员所需的思维习惯。与此同时，作者并非如空中楼阁一般高谈阔论，而是深入到具体的编程实践中，为读者指明向一流程序员迈步的具体措施。此外，软件构建牵扯到很多复杂的领域，作者难以在书中深入探讨，但他在相关章节后给出了更多的参考资源，为读者更进一步的了解和学习指出了方向。

完成一次优秀的软件构建需要关注方方面面的问题，大到确定需求和软件架构，小到选定变量名和控制语句，此外还包括版本管理、协同构建、测试等等问题，不一而足。这些问题都具有一定的复杂性，一流程序员能够顺利解决各式各样的问题，高效地构建出优质的软件。要达成这一目标，核心就是降低软件构建过程中的复杂性，以免陷入复杂的泥沼，导致项目超时延期，bug频出，甚至于构建失败，项目流产。

对一本书进行总结概括需要找到恰当的出发点，与对软件整体架构进行描述如出一辙。对于《代码大全》这本书来说，我认为可以用三条理念来总体把握，即抽象是应对复杂性的核心利器，始终面向问题来进行软件构建，以及，软件构建是一项人的协作活动。用这三条理念来指导和审视自己的代码构建的整个过程，相信能够让我的代码构建质量更上一层楼。

抽象是应对复杂性的核心利器

CPU只能识别用0和1代表高低电平变化的机器语言，我们为满足需求而构建的软件都必须将输入数据和控制信号转换为机器语言，并将CPU输出的01数据流转化为字符串或图像等便于理解的形式。在这个过程中有着从机器语言到汇编语言，再到高级语言的抽象递进与展开，我们才得以在更贴近人类自然表达的高级语言的层面进行软件构建工作。这就是抽象的强大，但可靠实用的抽象建立在展开时的复杂性可以被轻易完好解决的基础之上，如果我们在高级语言层面完成了软件构建，还要亲自将高级语言程序转换为汇编语言程序甚至于机器语言程序，那现在动辄几百MB大小的软件是难以顺利完成构建的，是编译器、汇编器、解释器等这些软件将我们从这些复杂的工作中解放出来。上面这个例子也向我们完美展示了计算机在解决复杂性方面的优势——将重复且繁重的工作抽象出来并转化为软件，再交由精确且迅速的CPU来应付这些问题。

虽然从高级语言到机器语言这之间的转化过程已经被很好地解决，但是从现实需求到高级语言之间的抽象与展开过程就无法那么轻松自在地被解决了。虽然随着编程和AI技术的发展，有着越来越多的重复且繁重的软件构建工作被发现，并进一步被人们研究和解决，如面向对象编程，敏捷开发乃至AI编程等。但是面对复杂多变的现实世界与五花八门的项目需求，总有一些需求和工作（这个比例并不小）必须交由程序员来应对，而从这些工作之中能够明显看出程序员间的水平差距。一流程序员能够对项目有足够的了解与把握，能够运用工具在需求与编程语言之间建立起清晰健壮的抽象关系，并且能够采取多种措施来实现和维护这种抽象关系，使项目一直处于控制当中，而不是滑向复杂的深渊。

抽象是强大的，但抽象本身会带来一定的复杂性，不恰当的抽象反倒会起反作用。就比如在日心说诞生之前，以地心说为基础来描绘宇宙星体的运行轨迹是一项十分困难且无法自洽的工作，而在日心说及牛顿的万有引力定律诞生后，人们可以轻松描绘出宇宙天体之间的位置关系及运动轨迹。正确与错误的抽象之间的差距可见一斑，所以要慎重对待抽象工作，不同抽象层之间的界限要足够明晰，要设计良好的类接口与子程序接口，更多采取防御式编程，否则可能会导致后期花费大量时间和精力在代码增删改查之上。
